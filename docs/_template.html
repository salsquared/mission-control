Headings

Levels

Each <h2> section is a collapsible <details class="section">, auto-numbered to match the ToC (CSS counter — reordering never desyncs it), with content in a <div class="section-body"> grey box. Every section holds at least one named <h3> subsection — auto-numbered X.Y (you write only the name; CSS supplies the number), so even a one-part section is always referenceable (this one is 1.1). Headings are Sora 800 / UPPERCASE; hierarchy is color + size, not weight. The <h3> above is green; below is an <h4> (teal).

Lists

Indexable markers

List markers are indexable by nesting depth so any item can be referenced by id instead of counted (e.g. "see b.ii.2"):

  • depth 1 → a, b, c
  • nesting deeper
    • depth 2 → i, ii, iii
    • and deeper still
      • depth 3 → 1, 2, 3
      • depth 4 → A, B, C
  • back to depth 1

Diagrams (Mermaid)

Conventions

Every node-based graph (architecture, flowchart, dependency graph, state machine, sequence, ER) is Mermaid — never ASCII art. Rendered diagrams are pan + zoom-enabled (drag to pan, scroll or the on-diagram buttons to zoom; it opens fit-to-view). Rules:

  • Wrap in <pre class="mermaid">. The init <script> at the bottom of this file themes it to the palette — copy it into every doc that has a diagram.
  • Line breaks in a label use <br/>, never \n (it renders literally).
  • Parens inside an edge label (pipe-delimited) must be quoted: |"probe()"|. Inside a quoted node label (["text()"]) they're fine.
  • Declare a direction first (TB for hierarchy/layers, LR for pipelines/flows) and keep it consistent.
  • Short stable lowerCamel node IDs + bracketed human labels; reuse the ID to draw more edges.
  • Don't hand-set node colors — let the shared theme own color.
Never put a Mermaid edge arrow (or any -- run) inside an HTML comment — it closes the <!-- --> early and dumps raw markup. Author diagrams only in real <pre class="mermaid"> blocks like the one below.

Tasks & to-dos

Authoring

Use <details class="task"> for any to-do list. They're collapsible and carry a checkbox + id. Mark done by adding checked to the <input> — the title strikes through and the id dims. Nest sub-tasks by putting another <details class="task"> inside .task-body; the id + checkbox color shifts per depth (blue → green → teal → sky → lime).

Ids are P<phase>.<task>.<sub>… — one dot per level, no depth cap: P1 → P1.2 → P1.2.3 → P1.2.3.4.5.
